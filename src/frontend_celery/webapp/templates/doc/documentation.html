User roles

There are four types of users in HerediVar:

  1. Superuser
  2. Standard user
  3. Read only user
  4. Anonymous user
An anonymous user can not view or edit any variant information on HerediVar. Anonymous users must ask a superuser to create an account for them. As the name implies, the read only user can only view information. He can neither insert variants nor classify them. The standard user is allwed to view and edit information on HerediVar and the superuser has access to the admin dashboard which allws him to import variants from HerediCare, reannotate all variants at once and create users.

Create and modify users

Create a new user

Only users with the "superuser" role are allowed to create new users. There is no possibility to self-register in HerediVar. A superuser can go here to create a new user account. This link will redirect to "Keycloak" which is the authentication platform of HerediVar. First click the "add user" button:

Simply fill out the form which is presented to you. Keep in mind that the username can not be changed later. You should select the "Update Password" and "Verify Email" user actions. It should look like this:
Next, you have to add the affiliation of the person. This is very important as the new user can not log into HerediVar if the affiliation is missing. To add the affiliation you can switch to the "Attributes" tab and enter "affiliation" in the key column and the actual affiliation in the value column. Click "Add" and "Save". It should look like this once you are finished with this step:
Now you have to set the password of the new user. For this switch to the "Credentials" tab. Simply insert a new password in "Reset Password" form. And enable the "Temporary" switch:
The last thing to do is to add a user role to the new user. Simply go to the "Role Mappings" tab, select either read_only, user or super_user to add privileges for the read only, standard user or superuser, respectively. It is sufficient to add the most general role to a user as privileges from more restricted roles are inherited by other roles. For example: If you want to create a new superuser only add the "super_user" role. You do not have to add the "user" and "read_only" roles, too.
